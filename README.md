# Telegram to VK notifier

Python-скрипт, который работает как Telegram userbot и отправляет уведомление в личные сообщения VK при получении нового личного сообщения от Telegram-контакта.

## Что делает

- авторизуется в Telegram через `Telethon`
- умеет входить по номеру телефона или через QR
- слушает новые сообщения
- обрабатывает только входящие личные сообщения
- отправляет уведомление только если отправитель является контактом или взаимным контактом
- отправляет уведомление в VK через `messages.send`
- отправляет имя контакта и текст сообщения
- для сообщений без текста пишет тип сообщения: `фото`, `видео`, `кружочек`, `голосовое`, `аудио`, `стикер`, `gif`, `документ`, `контакт`, `геопозиция`, `место`, `опрос`, `кубик`, `игра`, `счет`, `ссылка`
- для служебных событий пишет понятные подписи: `звонок`, `закрепленное сообщение`, `приглашение в чат` и другие

## Установка

Установите зависимости:

```bash
pip install -r requirements.txt
```

## Настройка

1. Скопируйте `.env.example` в `.env`.
2. Заполните значения в `.env`.

Пример:

```env
TG_API_ID=12345678
TG_API_HASH=your_telegram_api_hash
VK_TOKEN=your_vk_group_token
VK_USER_ID=123456789
TG_AUTH_MODE=qr
PROXY_HOST=127.0.0.1
PROXY_PORT=10808
DEBUG_LOG=1
```

## Параметры

- `TG_API_ID` - `api_id` приложения Telegram
- `TG_API_HASH` - `api_hash` приложения Telegram
- `VK_TOKEN` - токен сообщества VK с правом `messages`
- `VK_USER_ID` - числовой ID пользователя VK, которому отправлять уведомления
- `TG_AUTH_MODE` - режим входа: `phone` или `qr`
- `PROXY_HOST` - хост SOCKS5-прокси, если нужен
- `PROXY_PORT` - порт SOCKS5-прокси
- `PROXY_USERNAME` - логин прокси, если нужен
- `PROXY_PASSWORD` - пароль прокси, если нужен
- `DEBUG_LOG` - `1` для включения отладочных сообщений, `0` для отключения

## Где взять данные

### Telegram API

1. Перейдите на `https://my.telegram.org`.
2. Откройте `API development tools`.
3. Создайте приложение.
4. Скопируйте `api_id` и `api_hash`.

### VK token

1. Откройте нужное сообщество VK.
2. Перейдите в `Управление` -> `Работа с API` -> `Ключи доступа`.
3. Создайте ключ с правом `messages`.
4. Включите `Сообщения сообщества`.
5. Напишите этому сообществу с личного аккаунта хотя бы одно сообщение.

## Запуск

```bash
python tg_to_vk_userbot.py
```

## Авторизация Telegram

### Вариант 1: по номеру телефона

Укажите:

```env
TG_AUTH_MODE=phone
```

Скрипт запросит номер и код подтверждения.

### Вариант 2: через QR

Укажите:

```env
TG_AUTH_MODE=qr
```

Скрипт покажет QR-код в консоли. Далее в Telegram откройте:

`Настройки` -> `Устройства` -> `Подключить устройство`

Если включена двухэтапная аутентификация, скрипт дополнительно попросит пароль.

## Примечания

- Сессия `Telethon` сохраняется локально в файле `*.session`.
- Эти файлы уже добавлены в `.gitignore`.
- Если используется публичный репозиторий, не публикуйте `.env`.
- Если VK возвращает ошибку `901`, значит пользователь еще не разрешил сообщения от сообщества.
